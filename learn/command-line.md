# Command Line

The command line is a text-based interface for interacting with your computer. Instead of clicking through menus and folders, you type commands to navigate files, run programs, and automate tasks. It's how most developer tools — including Git, Python, and Claude Code — are operated.

For clinician-researchers, the command line is a prerequisite. Most AI-assisted research tools don't have graphical interfaces; they expect you to type a command in a terminal and read the output. Basic fluency — navigating directories, running scripts, reading error messages — unlocks everything else in CRAFT. You don't need to become a systems administrator, but you do need to be comfortable enough to follow instructions and troubleshoot when something doesn't work.

On macOS and Linux, the default shell is zsh or bash. On Windows, you'll typically use PowerShell or Git Bash. The core concepts (directories, files, commands, arguments) are the same across all of them.

## Getting Started

Your computer already has a terminal application — you just need to open it.

| Platform | Application | Notes |
|---|---|---|
| macOS | Terminal (built-in) | Found in Applications > Utilities. Also available: [iTerm2](https://iterm2.com), a more feature-rich alternative |
| Windows | [Windows Terminal](https://aka.ms/terminal) | Microsoft's modern terminal. Supports PowerShell, CMD, and WSL. Recommended over the default Command Prompt |
| Linux | Terminal (built-in) | Name varies by distribution — usually GNOME Terminal, Konsole, or xterm |
